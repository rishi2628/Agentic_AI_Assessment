ACME CORPORATION — IT SECURITY POLICY
Document:  VPN Access Policy
Version:   1.8
Effective: March 1, 2025
Owner:     Network Operations Team
Review:    Annually (next review: March 2026)
Classification: Internal

────────────────────────────────────────────────────────────────────
1. PURPOSE
────────────────────────────────────────────────────────────────────
This policy governs the use of ACME's Virtual Private Network (VPN)
to ensure secure remote access to corporate resources while protecting
the confidentiality and integrity of company data.

────────────────────────────────────────────────────────────────────
2. SCOPE
────────────────────────────────────────────────────────────────────
All employees, contractors, and third-party vendors who require remote
access to ACME internal systems.

────────────────────────────────────────────────────────────────────
3. APPROVED VPN CLIENT & ENDPOINT
────────────────────────────────────────────────────────────────────
   • Client software:  Cisco AnyConnect Secure Mobility Client v4.10+
                       (download from https://apps.company.com/vpn)
   • VPN endpoint:     vpn.company.com
   • Backup endpoint:  vpn-backup.company.com (use only if primary fails)
   • Protocol:         TLS 1.3 (IKEv2 not supported externally)

────────────────────────────────────────────────────────────────────
4. AUTHENTICATION
────────────────────────────────────────────────────────────────────
4.1 Required factors
   1. Active Directory (AD) credentials (username@company.com + password)
   2. MFA: Duo Security push notification or TOTP code
      (see MFA Policy v3.1 for supported methods)

4.2 Session management
   • Session timeout:     8 hours idle disconnect
   • Certificate validity: VPN certificates renewed automatically; if
     certificate error appears, re-download the client profile from
     https://apps.company.com/vpn.

────────────────────────────────────────────────────────────────────
5. USAGE RULES
────────────────────────────────────────────────────────────────────
5.1 Permitted uses
   • Accessing internal applications, file shares, and databases when
     working off-site.
   • Attending internal-only video calls or accessing intranet.

5.2 Restrictions
   • Split tunnelling is DISABLED — all traffic routed through corporate
     network for security monitoring.
   • VPN must NOT be used on untrusted public Wi-Fi without first
     connecting to a personal mobile hotspot or using WPA3 networks.
   • VPN access from embargoed countries requires prior Security team
     approval (submit via ServiceNow "VPN Geo-Exception" form).

5.3 Prohibited
   • Sharing VPN credentials or sessions with any other person.
   • Using VPN to route personal traffic or bypass geo-restrictions for
     non-business content.
   • Installing alternative VPN clients (WireGuard, OpenVPN) on
     company-managed devices without IT approval.

────────────────────────────────────────────────────────────────────
6. TROUBLESHOOTING GUIDE
────────────────────────────────────────────────────────────────────
6.1 Common issues (individual user)
   Issue: "Certificate Untrusted"
   → Re-download AnyConnect profile: https://apps.company.com/vpn
   → If persistent, raise ticket at help.company.com.

   Issue: MFA push not received
   → Wait 60 seconds and retry; ensure mobile data/Wi-Fi active.
   → Use TOTP code from Duo or Microsoft Authenticator app instead.

   Issue: Authentication fails after correct password
   → Check if account is locked (see Password Reset Policy §6).
   → Have your manager open a P3 ticket for AD synchronisation issue.

   Issue: Slow VPN
   → Connect to backup endpoint: vpn-backup.company.com
   → Run: Diagnostics → Send to IT (button in AnyConnect client)

6.2 VPN client log location
   • Windows: %APPDATA%\Cisco\Cisco AnyConnect Secure Mobility Client\
   • macOS:   ~/Library/Logs/Cisco AnyConnect/

6.3 Bulk / widespread outage (≥10 users affected)
   • Severity: P1 or P2 (see Incident Response Procedure §4)
   • Escalate immediately to: Network Operations on-call
     PagerDuty alias: @netops-oncall
     Email: netops@company.com
   • Network Ops will check:
       1. Cisco ASA / FTD firewall logs
       2. Duo Security admin portal for MFA service health
       3. Active Directory authentication service (LDAP port 636)
       4. DNS resolution for vpn.company.com
       5. BGP routing and upstream ISP status

6.4 Diagnostic commands (Network Ops use only)
   # Check active VPN sessions
   show vpn-sessiondb anyconnect

   # Check certificate expiry on ASA
   show crypto ca certificates

   # Verify AD/LDAP connectivity
   Test-NetConnection -ComputerName dc01.company.local -Port 636

────────────────────────────────────────────────────────────────────
7. REQUESTING VPN ACCESS
────────────────────────────────────────────────────────────────────
   • New access:     Manager submits ServiceNow "VPN Access Request"
   • Revocation:     HR/Manager submits "Offboarding" ticket; access
                     removed within 4 hours of ticket approval.
   • Privileged VPN: Requires CISO approval + annual recertification.

────────────────────────────────────────────────────────────────────
8. ENFORCEMENT
────────────────────────────────────────────────────────────────────
VPN usage is logged and monitored. Violations escalated to the SOC and
HR Business Partner for disciplinary review.
